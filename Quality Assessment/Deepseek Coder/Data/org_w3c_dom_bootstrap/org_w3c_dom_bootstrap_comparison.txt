Sure, here are some useful APIs from the `org.w3c.dom.bootstrap.DOMImplementationRegistry` package:

1. `DOMImplementation getDOMImplementation(String implementationName)`: This method returns a DOMImplementation object that implements the named DOM specification. The implementationName parameter is the name of the DOM specification to be implemented.

2. `DOMImplementationRegistry getInstance()`: This method returns a DOMImplementationRegistry object. This is the only way to obtain a DOMImplementationRegistry object.

3. `DOMImplementationList getImplementations(String feature)`: This method returns a DOMImplementationList of DOMImplementation objects that support the specified feature. The feature parameter is the name of the feature to be supported.

4. `boolean isDOMImplementationRegistered(String domain)`: This method returns a boolean indicating whether the specified DOM implementation is registered. The domain parameter is the name of the DOM implementation to be checked.

5. `boolean isDOMFeatureSupported(String domain, String feature)`: This method returns a boolean indicating whether the specified DOM feature is supported by the specified DOM implementation. The domain parameter is the name of the DOM implementation, and the feature parameter is the name of the feature to be checked.

Please note that these are the APIs from the `org.w3c.dom.bootstrap.DOMImplementationRegistry` package, and they are not inherent from its parent class.
