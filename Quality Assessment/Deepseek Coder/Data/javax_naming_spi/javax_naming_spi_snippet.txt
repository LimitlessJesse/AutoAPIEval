The `getInitialContext(Hashtable<?,?> environment)` method from `javax.naming.spi.NamingManager` is used to get an initial context. The initial context is the root of the naming context hierarchy.

Here is a complete code snippet:

```java
import javax.naming.spi.NamingManager;
import java.util.Hashtable;
import javax.naming.Context;

public class Main {
    public static void main(String[] args) {
        try {
            Hashtable<String, String> env = new Hashtable<>();
            env.put(Context.INITIAL_CONTEXT_FACTORY, "com.sun.jndi.ldap.LdapCtxFactory");
            env.put(Context.PROVIDER_URL, "ldap://localhost:389/dc=example,dc=com");
            Context ctx = NamingManager.getInitialContext(env);
            System.out.println("Initial context obtained successfully");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

In this code snippet, we are using a simple LDAP server running on localhost with port 389. The `INITIAL_CONTEXT_FACTORY` is set to `"com.sun.jndi.ldap.LdapCtxFactory"` which is the factory class for creating LDAP context. The `PROVIDER_URL` is set to `"ldap://localhost:389/dc=example,dc=com"` which is the URL of the LDAP server.

Please replace the `INITIAL_CONTEXT_FACTORY` and `PROVIDER_URL` with your own values.

Note: This code snippet requires the `jndi` and `ldap` libraries. If you are using a different naming service, you may need to use a different `INITIAL_CONTEXT_FACTORY`.

Also, this code snippet is for illustrative purposes and may not work in your environment without modification.

Please ensure that you have the necessary permissions to access the naming service and that the server is running and accessible.

If you encounter any issues, please refer to the documentation for your specific naming service for more information.
